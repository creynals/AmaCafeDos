# 📁 Context - Documentos Directores

Este directorio contiene los documentos de contexto del proyecto **import-1776956320164-2m9x2n**.

## Propósito

Aquí se almacenan todos los documentos que SYNAPTIC usa para:
- Generar sugerencias de próximos pasos (Tab Roadmap)
- Entender el contexto y requisitos del proyecto
- Extraer tareas pendientes y prioridades

## Tipos de Documentos Recomendados

| Tipo | Descripción | Ejemplo |
|------|-------------|---------|
| **Requisitos** | Especificaciones del cliente o producto | `requisitos_v1.md` |
| **Arquitectura** | Decisiones de diseño técnico | `arquitectura_sistema.md` |
| **Guías** | Estándares y convenciones del proyecto | `guia_estilos.md` |
| **Notas** | Información de reuniones o decisiones | `notas_kickoff.md` |
| **Backlog** | Lista de features o tareas pendientes | `backlog.md` |

## Formato de Tareas

Para que SYNAPTIC detecte tareas automáticamente, usa estos formatos:

### Checkboxes (Recomendado)
```markdown
- [ ] Tarea pendiente
- [x] Tarea completada
- [~] Tarea en progreso
- [!] Tarea bloqueada
```

### Prefijos
```markdown
- TODO: Descripción de la tarea
- PENDING: Tarea pendiente
- WIP: Trabajo en progreso
- BLOCKED: Tarea bloqueada
```

### Prioridades
```markdown
- [ ] [HIGH] Tarea de alta prioridad
- [ ] [MEDIUM] Tarea de prioridad media
- [ ] [LOW] Tarea de baja prioridad
```

## Uso

1. Crea documentos `.md` en esta carpeta
2. SYNAPTIC los analizará automáticamente
3. Las sugerencias aparecerán en el Tab Roadmap
